1. Мета роботи
Метою лабораторної роботи було створення основи віконної програми для програмування графічних операцій за зразком графічного редактора. Програма має містити меню, полотно для малювання, стандартні діалоги та базові команди для роботи з графічними фігурами.

2. Обраний підхід до реалізації
Для реалізації використано мову Python і стандартну бібліотеку tkinter.

Обрано векторний підхід до побудови графіки:
- кожна намальована фігура зберігається як окремий запис у списку об’єктів;
- малюнок відновлюється повторним перемальовуванням усіх фігур на Canvas;
- такий спосіб зручний для подальшого розширення редактора.

3. Основні елементи програмної реалізації

3.1. Головний клас програми
Функціональність інкапсульовано в класі GraphicalEditor, який наслідується від tk.Tk.

Роль класу:
- створює головне вікно програми;
- формує меню та полотно;
- зберігає поточний стан редактора;
- обробляє події миші;
- виконує збереження та відкриття файлів.

3.2. Створення інтерфейсу
Інтерфейс складається з таких частин:
- меню File для файлових операцій;
- меню Edit для майбутніх операцій редагування;
- меню Figures для вибору типу фігури;
- меню Properties для налаштування параметрів малювання;
- Canvas для безпосереднього малювання;
- рядок стану внизу вікна.

Полотно Canvas є основним графічним елементом, на якому відображаються лінії, прямокутники та еліпси.

3.3. Меню File
Реалізовані команди:
- New file;
- Open...;
- Save;
- Save As...;
- Exit.

Особливості:
- перед створенням нового файлу або виходом перевіряються незбережені зміни;
- якщо малюнок змінено, користувач отримує запит на збереження;
- відкриття та збереження виконуються через стандартні діалоги tkinter.filedialog.

3.4. Меню Edit
У меню Edit додано заглушки для команд майбутнього розширення:
- Cut;
- Copy;
- Paste;
- Delete selected;
- Select All;
- Paste from File.

На цьому етапі ці команди виводять повідомлення Not yet available. Це відповідає вимозі лабораторної про створення каркаса програми з поступовим наповненням функціональності.

3.5. Меню Figures
Меню Figures дозволяє обрати активну фігуру для малювання:
- Line;
- Rectangle;
- Ellipse.

Вибір фігури зберігається у змінній current_figure. Саме вона визначає, який графічний примітив буде побудовано під час роботи мишкою.

3.6. Меню Properties
Меню Properties відповідає за параметри графічних об’єктів:
- Line color;
- Fill color;
- Line thickness;
- Clear fill.

Призначення цих команд:
- вибір кольору лінії через стандартний діалог вибору кольору;
- вибір кольору заливки для прямокутника й еліпса;
- зміна товщини лінії через діалог введення числа;
- скидання заливки.

3.7. Події миші та алгоритм малювання
Малювання реалізовано через обробку подій:
- ButtonPress-1;
- B1-Motion;
- ButtonRelease-1.

Порядок роботи:
- при натисканні лівої кнопки миші фіксується початкова точка;
- при русі миші з натисненою кнопкою оновлюються координати попереднього перегляду;
- при відпусканні кнопки малюнок фіксується як повноцінна фігура.

Цей механізм використано для всіх трьох типів фігур:
- Line;
- Rectangle;
- Ellipse.

3.8. Модель збереження фігур
У програмі використано список shapes, у якому кожна фігура зберігається як словник з такими полями:
- type;
- x1;
- y1;
- x2;
- y2;
- outline;
- fill;
- width.

Такий формат дозволяє:
- легко зберігати малюнок у файл;
- коректно відновлювати його під час відкриття;
- масштабувати програму в майбутньому.

3.9. Формат файлів
Для Save і Open використано JSON-файли.

Структура файлу містить:
- службове поле format;
- номер версії;
- список shapes.

Переваги JSON у цій роботі:
- простий текстовий формат;
- зручний для перевірки викладачем;
- легко читати і відновлювати дані програми.

3.10. Перевірка коректності даних
Під час відкриття файлу виконується перевірка структури даних:
- чи є shapes списком;
- чи містить кожен елемент допустимий тип фігури;
- чи можна перетворити координати та товщину до чисел.

Це зменшує ризик помилок при відкритті пошкоджених або несумісних файлів.

4. Виконання вимог лабораторної роботи
Реалізовано такі пункти завдання:
- побудовано основу віконної програми;
- складено перелік базових графічних і допоміжних операцій у меню;
- запрограмовано малювання стандартних фігур;
- реалізовано стандартні діалоги;
- обрано векторний спосіб представлення графіки;
- реалізовано малювання лінії, прямокутника та еліпса за рухом мишки;
- реалізовано Save, Save As..., Open..., New file та Exit.

5. Особливості запуску
Програма використовує tkinter. У цій системі коректний запуск забезпечує системний Python:
- /usr/bin/python3 lab8/graphical_editor.py

У деяких збірках Homebrew Python може бути відсутній модуль _tkinter, тому для запуску GUI потрібен інтерпретатор із підтримкою Tk.

6. Що можна розширити надалі
Програма має каркас, придатний для подальшого розвитку. Наступні можливі вдосконалення:
- реалізація виділення фігур;
- повноцінні команди Cut, Copy, Paste, Delete selected;
- збереження у додаткових форматах;
- експорт у растрове зображення;
- додавання панелі інструментів.

Висновок
У лабораторній роботі №8 створено основу графічного редактора з використанням tkinter. Реалізовано головне вікно, меню, полотно, керування параметрами фігур, малювання лінії, прямокутника та еліпса, а також збереження й відкриття малюнка у форматі JSON. Програма побудована за векторним принципом і має структуру, придатну для подальшого розширення.
