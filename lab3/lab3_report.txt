2. Зауваження щодо середовища виконання
Локальна ОС: macOS.
Для відтворення Windows-орієнтованої частини лабораторної використано:
- GitHub Actions (`windows-latest`) для підготовки/перевірки артефактів
- vDos (`lab3/vDOS_for_Win64/vDos.exe`) для запуску COM у 64-бітному Windows-середовищі

3. Виконання пунктів розділу "Перевірка моделі виконання COM-програми"

Пункт 1.
Підготовлено BAT-файл `lab3_step1_notepad.bat` для запуску системної програми:
- використано `Start notepad.exe`, щоб блокнот запускався в окремому потоці.

Пункт 2.
Підготовлено C#-програму `lab3_step2_start_bat.cs`, яка моделює виклик BAT-файлу:
- запуск через `System.Diagnostics.Process.Start(...)`.

Пункт 3.
Підготовлено C#-програму `lab3_step3_build_com.cs`, яка:
- формує масиви байтів (hex-коди з методичних матеріалів),
- записує їх у двійкові файли через `BinaryWriter`,
- створює:
  - `lab3_message.com` (оригінальне повідомлення),
  - `lab3_message_long.com` (довше повідомлення).

Пункт 4.
Підготовлено BAT-файл `lab3_step4_run_com.bat` для запуску COM-файлу.
У 64-бітному середовищі запуск виконано через vDos.

Пункт 5.
Змодельовано повторний виклик BAT-файлу з C# (аналог п.2) для COM-сценарію.
Очікуване повідомлення при коректній побудові COM:
`Testing: print a line:`

Пункт 6.
Підготовлено варіант з довшим текстом:
- `lab3_message_long.com`
- у байт-коді змінено довжину переходу `JMP MAINPROG` (після `EB`) під нову довжину рядка.

4. Перевірка в CI (GitHub Actions)
Додано workflow:
- `.github/workflows/lab3-com-model.yml`

Що перевіряється автоматично:
- компіляція C# утиліт кроків 2 і 3,
- генерація обох COM-файлів,
- контроль розмірів (короткий 39 байт, довгий 55 байт),
- публікація артефактів.

5. Матеріали, які треба додати у фінальну версію звіту (Word/PDF)
Згідно вимог, обов'язково додати знімки екрана:
- запуск `lab3_step1_notepad.bat`,
- запуск `list1` у vDos,

Висновок
У лабораторній роботі №3 підготовлено та перевірено модель виконання COM-програми: виклик BAT-файлів, запуск з C#, формування COM-файлів з двійкових кодів і модифікацію повідомлення через ручну зміну байт-коду. Для сумісності з 64-бітним Windows передбачено запуск через емулятор vDos.
