Тема
Розширення графічного редактора: скасування операцій, історія змін, виокремлення, копіювання та пересування фігур.

1. Мета роботи
Метою лабораторної роботи було вдосконалити реалізований у ЛР8 графічний редактор і додати:
- скасування поточної операції малювання клавішею Esc;
- скасування вже виконаних операцій (Undo);
- виокремлення, копіювання, вставку та пересування фігур за допомогою миші.

2. Вихідна основа
За основу використано програму з ЛР8:
- файл реалізації: lab8/graphical_editor.py;
- технологія: Python + tkinter;
- модель графіки: векторна (збереження фігур як списку об’єктів).

3. Обраний підхід
Використано векторний підхід до зберігання та обробки малюнка:
- кожна фігура зберігається у списку shapes;
- елементи списку містять параметри фігури (тип, координати, колір, товщина);
- операції Undo реалізовано через історію станів history;
- після змін полотно перемальовується на основі поточного вмісту shapes.

Такий підхід є простим, наочним і добре підходить для реалізації скасування, копіювання та переміщення у редакторі базового рівня.

4. Алгоритми і програмна реалізація

4.1. Скасування поточної операції (Esc)
Завдання: користувач може перервати незавершену дію до її фіксації.

Алгоритм:
1. Перехопити натискання клавіші Esc.
2. Якщо виконується малювання фігури:
- видалити тимчасовий preview-об’єкт;
- скинути службові змінні режиму малювання.
3. Якщо виконується виокремлення:
- прибрати тимчасову рамку виокремлення;
- скинути стан виділення (або залишити попередній, залежно від контексту).
4. Якщо виконується Paste mode:
- видалити тимчасові preview-копії;
- вийти з режиму вставки.
5. Якщо виконується пересування виділених фігур:
- повернути фігури до початкових координат;
- завершити режим пересування без фіксації змін.

Результат:
- незавершена операція повністю скасовується, малюнок повертається до консистентного стану.

4.2. Undo (скасування останньої зафіксованої операції)
Завдання: скасувати вже виконану дію (малювання, видалення, пересування, вставку).

Алгоритм:
1. Перед кожною операцією, що змінює shapes, зберегти глибоку копію поточного стану у history.
2. При команді Undo:
- перевірити, чи history не порожній;
- взяти останній збережений стан;
- замінити поточний shapes цим станом;
- перемалювати полотно.
3. Оновити службовий стан редактора (selection/status/dirty flag).

Реалізація:
- команда меню Edit -> Undo;
- гарячі клавіші Ctrl+Z та Command+Z.

Результат:
- користувач може покроково відкотити останні зміни.

4.3. Виокремлення фігур (Select)
Завдання: виділити одну або кілька фігур для подальших операцій.

Алгоритм:
1. Додати окремий режим роботи Select (у меню Figures).
2. При натисканні миші в режимі Select:
- якщо клік в межах вже виділеної області, перейти в режим пересування;
- інакше почати формування рамки виокремлення.
3. При русі миші оновлювати координати рамки виокремлення.
4. При відпусканні кнопки:
- визначити прямокутник виокремлення;
- знайти всі фігури, bounding box яких перетинається з цією рамкою;
- зберегти індекси вибраних фігур.
5. Намалювати пунктирний контур виділення.

Результат:
- можна виділити довільну групу фігур і виконувати над нею наступні дії.

4.4. Копіювання (Copy)
Завдання: зберегти виділені фігури у внутрішній буфер обміну.

Алгоритм:
1. Перевірити, що є виділені фігури.
2. Обчислити опорну точку виділення (верхній лівий кут selection bbox).
3. Для кожної виділеної фігури:
- створити копію;
- перевести координати у відносні щодо опорної точки.
4. Зберегти набір відносних фігур у clipboard_shapes.

Результат:
- отримано буфер, незалежний від абсолютних координат, зручний для багаторазової вставки.

4.5. Вставка (Paste) і позиціювання мишкою
Завдання: вставляти копії у довільне місце.

Алгоритм:
1. Команда Paste вмикає режим вставки (paste mode).
2. Під час руху миші відображати preview-копії у поточній позиції курсора.
3. При кліку миші:
- зберегти поточний стан у history;
- додати у shapes нові фігури як зміщені копії з clipboard_shapes;
- завершити paste mode;
- призначити нові фігури поточним виділенням.

Результат:
- вставка працює інтерактивно: користувач одразу бачить, куди буде розміщено копію.

4.6. Пересування виділених фігур мишкою
Завдання: переміщати раніше виділені фігури.

Алгоритм:
1. У режимі Select при натисканні всередині рамки виділення:
- зафіксувати стартову точку move_start;
- зберегти початковий стан shapes у move_origin_shapes.
2. Під час руху миші:
- обчислити зсув dx, dy;
- застосувати зсув до координат усіх вибраних фігур;
- оновити відображення.
3. При відпусканні кнопки:
- якщо положення змінилося, додати початковий стан у history;
- завершити режим пересування.
4. При Esc під час пересування:
- відновити move_origin_shapes;
- завершити пересування без фіксації змін.

Результат:
- реалізовано інтуїтивне переміщення виділених об’єктів із можливістю скасування.

4.7. Додаткові реалізовані операції
Окрім обов’язкових пунктів, додано:
- Delete selected: видалення виділених фігур;
- Cut: Copy + Delete selected;
- Select All: виділення всіх фігур.

Ці операції підвищують практичну цінність редактора і відповідають логіці меню графічних застосунків.

5. Структури даних і службові стани
У реалізації використано ключові структури:
- shapes: поточний список фігур на полотні;
- history: стек станів для Undo;
- selected_indices: індекси виділених фігур;
- selection_bbox: межі виділення;
- clipboard_shapes: буфер копій для Paste;
- preview-об’єкти для тимчасового показу (малювання, виокремлення, вставка).

Такий поділ даних забезпечує відокремлення постійного стану від тимчасового інтерфейсного стану.

6. Перевірка роботи
Проведено функціональну перевірку сценаріїв:
- малювання Line/Rectangle/Ellipse;
- скасування незавершеного малювання клавішею Esc;
- Undo після малювання, вставки, видалення, пересування;
- виокремлення рамкою, Copy/Paste, багаторазова вставка;
- пересування виділених фігур і скасування пересування через Esc;
- збереження і відновлення малюнка (Open/Save/Save As), New file, Exit.

7. Висновок
У лабораторній роботі №9 розширено функціональність редактора, створеного у ЛР8. Реалізовано скасування поточної операції, історію змін з Undo, виокремлення групи фігур, копіювання, вставку і пересування за допомогою миші. Додатково реалізовано Delete selected, Cut та Select All. Отриманий програмний результат відповідає вимогам завдання і може бути основою для подальшого розвитку графічного редактора.
