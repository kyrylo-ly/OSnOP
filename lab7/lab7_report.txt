1. Вихідні артефакти DLL
Для роботи використано бібліотеку з попередньої лабораторної:
- Lab5Dll.lib (файл імпорту)
- Lab5Dll.dll (динамічна бібліотека)

2. Використання DLL у програмі C#
Файл програми:
- lab7_csharp_dll_test.cs

Спосіб підключення:
- неявне зв'язування через механізм P/Invoke (`DllImport`)
- для кожної DLL-функції оголошено зовнішній метод з відповідною сигнатурою
- задано `CallingConvention.Cdecl`
- для BOOL використано маршалінг `UnmanagedType.Bool`

Перевірені функції:
- AnalyzeCpuLoadW
- CountStableTransitionsW
- EstimateDeliveryCostW
- BuildUserTagW
- EvaluatePasswordStrengthW

Результати C#:
- всі тести пройдено
- детальний консольний вивід наведено у файлі `lab7_csharp_output_with_comments.txt`

3. Використання DLL у програмі Python
Обрана мова:
- Python

Файл програми:
- lab7_python_dll_test.py

Спосіб підключення:
- модуль `ctypes`, завантаження бібліотеки через `ctypes.CDLL`
- для кожної функції вказано `argtypes` і `restype`
- тип WinAPI `BOOL` представлено як `ctypes.c_int`
- рядкові параметри `LPCSTR/LPSTR` обробляються через `bytes` та `create_string_buffer`

Перевірені функції:
- AnalyzeCpuLoadW
- CountStableTransitionsW
- EstimateDeliveryCostW
- BuildUserTagW
- EvaluatePasswordStrengthW

Результати Python:
- всі тести пройдено
- детальний консольний вивід наведено у файлі `lab7_python_output_with_comments.txt`

4. Пояснення щодо розміщення DLL
Для автоматичного пошуку `Lab5Dll.dll` файл бібліотеки слід розміщувати:
- поруч з виконуваним файлом програми, або
- у каталозі, що входить у системну змінну `PATH`.

5. Факультативні міркування щодо ефективного використання DLL в інтерпретованих мовах
У Python/Visual Basic/Java доцільно виносити в DLL:
- обчислювально інтенсивні процедури (математика, обробка сигналів, статистика);
- алгоритми пакетної обробки масивів чисел;
- функції перевірки даних і форматування, що часто викликаються;
- код, який уже реалізований у C/C++ та перевірений тестами.

Очікуваний ефект:
- підвищення продуктивності за рахунок виконання критичних ділянок у машинних кодах;
- повторне використання однієї DLL з різних мов;
- спрощення супроводу, оскільки бізнес-логіка централізується в одному модулі.
