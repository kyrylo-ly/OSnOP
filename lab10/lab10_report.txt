1. Мета роботи
Метою лабораторної роботи було завершити розробку графічного редактора, додавши:
- можливість перегляду та редагування великих за розміром малюнків за допомогою смуг прокрутки;
- правильне обчислення координат курсору при зміщенні полотна;
- імпорт растрових зображень на полотно;
- операції фільтрування зображень (Grayscale, Invert, Sepia);
- операції геометричного перетворення (Rotate 90, Flip Horizontal/Vertical);
- експорт вмісту полотна у растровий файл.

2. Вихідна основа
За основу використано програму з ЛР8 та ЛР9:
- файл реалізації: lab10/graphical_editor.py;
- технологія: Python + tkinter;
- додаткова бібліотека: Pillow (PIL) для попіксельної обробки зображень.

3. Обраний підхід
Для реалізації нових функцій використано комбінований підхід:
- Векторна модель розширена новим типом об'єкта "image", що зберігає растрову матрицю.
- Для роботи з великим полотном застосовано механізм `scrollregion` у tkinter.Canvas.
- Попіксельна фільтрація реалізована за допомогою методів бібліотеки Pillow (ImageOps) та прямого маніпулювання пікселями через матрицю кольорів.
- Для збереження зображень у JSON-файлі проєкту використано кодування Base64, що дозволяє тримати всі дані в одному текстовому файлі.

4. Алгоритми і програмна реалізація

4.1. Прокручування великих малюнків (Scrolling)
Завдання: реалізувати роботу з полотном, розмір якого перевищує розмір вікна.

Алгоритм:
1. Помістити `tk.Canvas` у контейнер `tk.Frame`.
2. Додати вертикальну та горизонтальну смуги прокрутки (`tk.Scrollbar`).
3. Встановити параметр `scrollregion` полотна рівним 3000x3000 пікселів.
4. Зв'язати команди прокрутки смуг з методами `xview` та `yview` полотна.

Результат:
- Користувач може вільно пересуватися по великому малюнку.

4.2. Корекція координат курсору
Завдання: забезпечити точне малювання та виокремлення фігур при будь-якому положенні смуг прокрутки.

Алгоритм:
1. У всіх обробниках подій миші (`on_mouse_down`, `on_mouse_move` тощо) замінити використання сирих координат `event.x` та `event.y`.
2. Використовувати методи полотна `self.canvas.canvasx(event.x)` та `self.canvas.canvasy(event.y)`.
3. Отримані значення використовувати як абсолютні координати у системі координат полотна.

Результат:
- Малювання та вибір об'єктів працюють без похибок незалежно від прокрутки.

4.3. Фільтрування зображень
Завдання: реалізувати математичні перетворення кольорів пікселів для створення спецэффектів.

Алгоритми фільтрів:
- Grayscale: Кожен піксель перетворюється на відтінок сірого за формулою яскравості (Luminance).
- Invert: Значення кожного каналу (R, G, B) замінюється на (255 - значення).
- Sepia: Застосовується трансформаційна матриця для кожного пікселя:
  R' = 0.393R + 0.769G + 0.189B
  G' = 0.349R + 0.686G + 0.168B
  B' = 0.272R + 0.534G + 0.131B

Реалізація:
- Створено меню "Filters", операції якого застосовуються до вибраного об'єкта типу "image".

4.4. Геометричні перетворення
Завдання: реалізувати поворот та віддзеркалення зображень.

Реалізація:
- Rotate 90°: Використання методу `img.rotate(angle, expand=True)` для зміни орієнтації зі збереженням пропорцій.
- Flip: Використання `ImageOps.mirror` (горизонтально) та `ImageOps.flip` (вертикально).
- Після кожного перетворення bounding box об'єкта перераховується для коректного виділення.

4.5. Експорт у растр
Завдання: зберегти поточний стан редактора як готове зображення.

Алгоритм:
1. Використати модуль `PIL.ImageGrab` для захоплення області екрана, де розташоване полотно.
2. Визначити координати кореня вікна та розміри Canvas.
3. Зберегти отримане зображення у форматі PNG або JPG за вибором користувача.

5. Структури даних
- `SUPPORTED_FIGURES`: додано "image".
- `_photo_cache`: список для зберігання посилань на об'єкти `ImageTk.PhotoImage` (запобігає видаленню зображень з пам'яті збирачем сміття).
- `pil_image_b64`: поле у JSON-структурі для збереження растрових даних у текстовому форматі.

Проведено тестування:

- Імпорт фотографій різних форматів.
- Застосування всіх фільтрів до імпортованих фото.
- Повороти зображень на 90 та -90 градусів.
- Збереження проєкту з картинками у JSON та його успішне відкриття.
- Експорт композиції з векторних фігур та фото у файл .png.
