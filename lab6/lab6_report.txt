- Lab5Dll.lib (файл імпорту)
- Lab5Dll.dll (динамічна бібліотека)

2. Програма тестування неявного зв’язування
Файл програми:
- lab6_implicit_linking_test.cpp

Особливості реалізації:
- підключено заголовочний файл Lab5Dll.h
- налаштовано неявне зв’язування з import-library
- викликано всі експортовані функції DLL
- додано перевірки як коректних, так і частини некоректних вхідних даних

Тестовані функції:
- AnalyzeCpuLoadW
- CountStableTransitionsW
- EstimateDeliveryCostW
- BuildUserTagW
- EvaluatePasswordStrengthW

3. Результати виконання за методом неявного зв’язування
Файл з надрукованими результатами та коментарями:
- lab6_implicit_output_with_comments.txt

Консольний вивід:

Коментар:

- обчислені значення відповідають очікуваним

4. Програма тестування явного зв’язування
Файл програми:
- lab6_explicit_linking_test.cpp

Особливості реалізації:
- DLL завантажується під час виконання через LoadLibraryA
- адреси функцій отримуються через GetProcAddress
- виконано перевірку наявності бібліотеки і всіх необхідних експортів
- наприкінці виконання бібліотека коректно звільняється через FreeLibrary

Тестовані функції:
- AnalyzeCpuLoadW
- CountStableTransitionsW
- EstimateDeliveryCostW
- BuildUserTagW
- EvaluatePasswordStrengthW

5. Результати виконання за методом явного зв’язування
Файл з надрукованими результатами та коментарями:
- lab6_explicit_output_with_comments.txt

Консольний вивід:
Explicit linking: all DLL tests passed.
Loaded from: Lab5Dll.dll
avgLoad=59.175, peakIndex=2, cost=34.5984
tag=AL-0042, strongScore=100, weakScore=11

Коментар:
- тести підтвердили коректне завантаження DLL та виклики функцій через вказівники
- результати обчислень у явному та неявному режимах збігаються

6. Перевірка умов запуску (пункти 4-5 з завдання)
У CI workflow перевірено два режими:
- без DLL поруч із EXE (очікуване падіння implicit-клієнта)
- з DLL поруч із EXE (успішний запуск implicit та explicit клієнтів)

Висновок
У лабораторній роботі №6 реалізовано та перевірено два способи використання DLL у C++: неявне та явне зв’язування. Обидва підходи працюють коректно, всі функції DLL проходять тести, а результати виконання узгоджені між собою.
